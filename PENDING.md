# HackSense — Pending Work (2026-05-17)

## Project Purpose

HackSense is a React + Express web game built for the HackTudo 2025 Festival de Cultura Digital in Rio de Janeiro. Players are shown two images side-by-side — one generated by AI, one by a human — and must click the AI-generated one. Scores are saved to a Supabase-backed leaderboard. An admin panel protected by HTTP Basic Auth allows image pair uploads (compressed via Sharp) stored in Supabase Storage.

## Current State

- Frontend: React 18.3 + Vite 5.4, deployed to Vercel
- Backend: Serverless functions under `/api/` (Vercel functions) + an Express server (`server.js`) for local dev
- Storage: Supabase (Postgres for rankings/image metadata, Supabase Storage for images)
- Deployment: Vercel (hobby plan, limited to a small number of serverless functions)
- Branch is 16 commits behind `origin/master` — local master lags remote

## Prioritized Pending Items

1. **Hardcoded credentials in `supabase-direct.js`** — Supabase anon key is committed in plaintext. Should be read from `process.env.SUPABASE_URL` / `process.env.SUPABASE_ANON_KEY` with a fallback comment, matching Vercel env var conventions.

2. **Admin password hardcoded in `api/admin/_middleware.js`** — credentials (`hacksense2025` / `HackSense2025!`) are in source. Should move to `process.env.ADMIN_USERNAME` / `process.env.ADMIN_PASSWORD`.

3. **`server.js` starts unconditionally when `NODE_ENV !== 'production'`** but the check is wrong: in Vercel serverless `NODE_ENV` is `production`, so the `listen()` block is dead locally unless NODE_ENV is unset. The guard should check for `process.argv[1]` being the current module (i.e., run directly), which `import.meta.url` enables.

4. **`Game-optimized.jsx` checks `selectedImage.isAI` as the correct answer** but the game header still says "CLIQUE NA IMAGEM FEITA POR I.A." — yet the scoring comment says "+10 points por acerto". Need to verify the answer logic is consistent with what the UI tells the user to do (click the AI image, not the human one). Currently `isAI === true` → `isCorrect = true` → `+10` — this is correct per the UI prompt.

5. **No `.env.example` file** — contributors cannot set up the project without reading `supabase-direct.js` to find what environment variables are expected.

6. **README.md has AI attribution** — `🤖 Generated with [Claude Code](https://claude.com/claude-code)` footer violates operator attribution policy.

7. **CHANGELOG.md missing** — no changelog file exists.

8. **`docs/bmad/` missing** — BMAD v6 PRD and architecture docs not yet created in the `docs/bmad/` tree.

9. **`api/game-data.js` `cachedConfig`** is module-level state shared across serverless invocations, but Vercel cold-starts a new instance per function; `updateConfig` is exported but never called from anywhere. Config should be fetched dynamically from a Supabase table or env var.

10. **Email validation in `api/score.js`** uses `email.includes('@')` — too permissive; a basic regex would be more robust.
